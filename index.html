Binary search — CPIC template

Three ways to search a sorted list. Each one takes only $O(\log n)$ steps — way faster than checking elements one by one ($O(n)$). We keep two pointers $l$ and $r$ marking the part we haven't checked yet, and loop while $l \le r$. The highlighted line is the one running right now.

    How this works: $l$ and $r$ point to the start and end of the part we still need to check. We loop while $l \le r$ and look at the middle: $\text{mid} = \lfloor (l + r) / 2 \rfloor$. For lower_bound / upper_bound, when the middle value is "good enough" we save it as our best answer so far ($\text{ans} = \text{mid}$) and keep searching to the left to see if there's an even earlier one. Each search takes $O(\log n)$ steps.
    


    Handy facts: the number of values smaller than $x$ is just lower_bound(x), and the number of values larger than $x$ is $n - $ upper_bound(x).

Binary search the answer

  
Sometimes the thing you're looking for isn't in a list — it's a number in some range $[l, r]$. Pick a guess $x$, write a $\text{check}(x)$ function that says "yes that works" or "no that doesn't", and narrow down. This works as long as $\text{check}$ has a clean cutoff: once it flips from no to yes (or yes to no), it never flips back.

    How BSTA works: you have a range of possible answers, from $l$ to $r$. The function $\text{check}(x)$ tells you whether a guess $x$ is good. If there's a clean cutoff — some value $x^\star$ where everything on one side works and everything on the other side doesn't — you can find $x^\star$ in just $O(\log(r - l))$ tries. When $\text{check}(\text{mid})$ is true, save $\text{ans} = \text{mid}$ and shrink toward the better side; otherwise shrink the other way.
    


    Quick tip: if a problem asks for the smallest possible biggest ($\min \max$) or the biggest possible smallest ($\max \min$) of something, it's often a BSTA problem.
